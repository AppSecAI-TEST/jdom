 /**
 79   
  * This interface provides a base support for the {@link SAXOutputter}.
 80   
  * <p>
 81   
  * People who want to create a custom SAXOutputProcessor for SAXOutputter are
 82   
  * able to implement this interface with the following notes and restrictions:
 83   
  * <ol>
 84   
  * <li>The SAXOutputter will call one, and only one of the
 85   
  * <code>process(SAXTarget,Format,*)</code> methods each time the SAXOutputter
 86   
  * is requested to output some JDOM content. It is thus safe to assume that a
 87   
  * <code>process(SAXTarget,Format,*)</code> method can set up any infrastructure
 88   
  * needed to process the content, and that the SAXOutputter will not re-call
 89   
  * that method, or some other <code>process(SAXTarget,Format,*)</code> method
 90   
  * for the same output sequence.
 91   
  * <li>The process methods should be thread-safe and reentrant: The same
 92   
  * <code>process(SAXTarget,Format,*)</code> method may (will) be called
 93   
  * concurrently from different threads.
 94   
  * </ol>
 95   
  * <p>
 96   
  * The {@link AbstractSAXOutputProcessor} class is a full implementation of this
 97   
  * interface and is fully customisable. People who want a custom SAXOutputter
 98   
  * are encouraged to extend the AbstractSAXOutputProcessor rather than do a full
 99   
  * re-implementation of this interface.
 100   
  *
 101   
  * @see SAXOutputter
 102   
  * @see AbstractSAXOutputProcessor
 103   
  * @since JDOM2
 104
